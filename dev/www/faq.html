What to input in GUI?

What is Path Type

  1. IFS - Isilon File System

    2. The format for IFS path should be similar to the format provided in example.
    e.g. /ifs/L1/L2/L3/L4/L5/L6/L7

    L1 - Region

    • am
    • ap
    • eu

    L2 - Location

    • for am: hou,cal,rio
    • for ap: bng,pej,pth,bri
    • for eu: ams,ris

    L3 - Protection Type

    • backup
    • nobackup
    • scratch

    L4/L5 - Business Unit
    e.g. . ua/sepco, ua/sda etc

    L6 - Data Type

    • data
    • proj
    • seis
    • owdata
    • corpdata
    • field

    L7 - User Folder
    e.g. /ifs/am/hou/backup/ua/sepco/proj/< user folder >

  2. DFS - Distributed File System Format

    3. The format for IFS path should be similar to the format provided in example.
    e.g. . \\Region\tcs\Location\Business Unit\Data Type\<User Folder>

    Region

    • americas.shell.com
    • asia-pac.shell.com
    • europe.shell.com

    Location

    • for americas.shell.com: hou,cal,rio
    • for asia-pac.shell.com: bng,pej,pth,bri
    • for europe.shell.com: ams,ris

    Business Unit
    e.g. ua.sepco, ua.sda etc

    Data Type

    • data
    • proj
    • seis
    • owdata
    • corpdata
    • field

Note: Tool provides support to sub directories of level7 upto any level.

What is the Output

Output is sent to the requestor in the form of an email after 15 minutes of submitting the request.

Sample Email Output:

Message Interpretation

  1. No FGO Group Assigned or if the FGO group does not have required access

    2. No FGO group is associated with the requested storage path.

    What to do Further?

    In order to have access the following is the required.

    1. Create the required FGO group in the appropriate domain by submitting a request in ServiceNow:
      Active Directory - Group - Add Change Delete Rename (refer to snippet below)
    2. Add the user account(s) to the newly created FGO group by submitting a request in ServiceNow:
      Active Directory - Group - User - Add / Delete (refer to snippet below)
    3. Request for the newly created FGO group to the applied to the path by submitting a request in ServiceNow:
      TCS - Permission – Change (refer to snippet below)

      4. Then select Technical Storage and Data Services. Top navigation choices change once selected.

  2. Duplicate Request

    3. The request for this Path has already been submitted and its under process. Please wait, you will receive the information over email.
    Request for this path can be submitted again after you have received the information over email.

  3. Incorrect Path

    4. Please follow the naming standards as per the guidelines mentioned in section What to input in GUI?

  4. Incorrect Location or Incorrect Region or Not TCS path

    5. Please enter the correct location in the path. This Team can only provide details on the following locations:

    • \\europe.shell.com\tcs\ams
    • \\europe.shell.com\tcs\ris
    • \\amercias.shell.com\tcs\hou
    • \\amercias.shell.com\tcs\cal
    • \\asia-pac.shell.com\tcs\pej
    • \\asia-pac.shell.com\tcs\pth
    • \\asia-pac.shell.com\tcs\bng
    • \\americas.shell.com\tcs\rio
    • \\americas.shell.com\tcs\bri

How to get access if the path top level is Linux style

In order to have access the following is the required

  1. Have a valid Linux account.
    Linux (Global) account - Create – in ServiceNow here
  2. Their Linux account must be correctly linked to the user’s Windows account.
  3. The account must be a member of the Linux security group Linux_group
    Linux (Global) Group membership and management - in ServiceNow here

    4. Linux (Global) Group membership and management
